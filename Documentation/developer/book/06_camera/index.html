Coordinate Systems

Several coordinate systems are involved in taking a vertex from its local coordinate system to a pixel position within a window. The majority of these coordinate systems are controlled by the current camera, operating in conjunction with the geometrical transform objects.

A vertex is initially specified in local coordinates. This is an arbitrary coordinate system, chosen in a way that makes sense for the vertex's object. Typically the origin will be placed at the center of the object, to allow it to be transformed intuitively.

During rendering, vertices are transformed from their local coordinate system to world coordinates. This coordinate system is also arbitrary, however it establishes a common coordinate system for all objects within the world.

The camera position and orientation is typically specified in world coordinates, but for rendering purposes a camera object also needs a coordinate system known as view coordinates. This coordinate system moves the camera location to the origin, with the camera looking down the -z axis and the camera up vector placed along the +y axis.

Although the view coordinate system establishes the camera's overall view of the world, a further coordinate system is necessary to establish what falls within the camera's field of view. This coordinate system is known as frustum coordinates. This provides the camera with the "lens" through which it sees the world, and allows the camera to apply a particular type of projection (e.g., perspective or orthographic) to the scene.

Quesa's frustum coordinate system ranges from -1 to +1 in x and y, and 0 to -1 in z (where 0 is the near clipping plane, and -1 is the far clipping plane). Once the scene has been transformed to this coordinate system, it then passes through the view port transform. This transform selects a portion of the xy plane of the frustum coordinate system, and it is this portion which is mapped to the draw context for display.

Once the coordinates have passed through the view port transform, they can then be mapped to the draw context. This is achieved with the frustum to window transform. This transform takes the portion of the frustum that was selected with the view port transform, and maps it to the active pane of the draw context being rendered to.
